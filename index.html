Part 2: Bezier Surfaces with Separable 1D de Casteljau

Briefly explain how de Casteljau algorithm extends to Bezier surfaces and how you implemented it in order to evaluate Bezier surfaces.

The intuition behind our implementation for evaluating Bezier surfaces stems from the implementation of de Casteljau for Bezier curves. To extend from Bezier curves to surfaces, we perform a series of separable 1D operations. For the Bezier surface, we have a 2D (nxn) grid of control points, so we reduce from Bezier surface to curve by taking each row of control points and performing de casteljau on t=u (given parameter). We will then accumulate a point for each row and so then, we again perform de casteljau for these points treating it as a single row on t = v (the given parameter). This last operation of de casteljau will then result in an evaluated point on (u,v) on the bezier surface.

Section II: Triangle Meshes and Half-Edge Data Structure

Part 3: Area-Weighted Vertex Normals

Briefly explain how you implemented the area-weighted vertex normals.

We traverse the different faces of the triangles connected to this vertex starting at the halfedge. We get the vertex of this halfedge, then move to the next half edge using halfedge->next(). Again, we get the vertex associated with this half edge and move to the next half edge, where we can get the final vertex. Once all vertices are found, we can calculate the cross product and add this to our running cross product sum. By adding the cross product, we do not need to weigh the norm by the area, and can simply add all the cross products together. We then traverse to the next halfedge and repeat this process until all cross products have been summed together. Finally, we normalize this running sum to return our final answer.

Part 6: Loop Subdivision for Mesh Upsampling

Briefly explain how you implemented the loop subdivision and describe any interesting implementation / debugging tricks you have used.

The initial step of our implementation was to compute the positions of both the new and old vertices; we perform this step before subdividing so that it's easier to traverse the entire mesh. For each new vertex, we compute the new position by weighting the sum of the four connected vertices (one opposing pair is multiplied by 3/8 and the other 1/8). Then we update the new position of the old vertex by (1-n*u) * original_position + (u * original_neighbor_position_sum), where n is the degree of the vertex, u is 3/16 if n==3 else 3/8n, and original_neighbor_position_sum is the sum of positions of its neighbouring vertices. Once we compute all the new positions, we split all the current edges and for each edge that's connected to a new and old vertex, we will flip these edges. Finally, we update the original positions of all vertices to its new positions.

We can make the cube subdivide more symmetrically by preprocessing (splitting the edge of each face of the cube). This achieves a more balanced connectivity of the mesh (more vertices and edges distributed evenly). Essentially, the intuition behind this is that by having a stronger uniform foundation initially, the latter subdivisions will follow this topology. If we had an extremely uneven topology, the subdivisions will strengthen this structure. So by preprocessing, adding splits and flips initially, we build a more symmetrical, balanced structure that will then be symmetrically connected as we increase the amount of subdivisions.
